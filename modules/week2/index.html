# The shell

Shell is the interpreter for the command line interface. Unix and therefore, macOS, has many available shells available. Each have a differente story and set of features. All of them share the same set of commands and basic functionality. They have names like Bourne, Bash, fish, C, csh, tcsh and so on. When open your Teminal to use the CLI, you are in a shell.

macOS Mojave and previous versions used Bash as their default shell, but in 2019 Apple decided to change that and set the Z shell as their new default.

Z shell (or just Zsh) , is an extended version of the Bourne shell (Bash), with plenty of new features, and support for plugins and themes. Since it’s based on the same shell as Bash, Zsh has many of the same features.

# Environment variables in macOS

An environment (ENV) variable is a variable that you create, just like in JavaScript, which can be read and used by the Operating System. Every OS has ENV variables.

These variables can be accessed, not just by the OS but also by other programs running on the OS. They can be used to share settings and information between programs. Useful commands to use are export, echo and printenv.

  • export makes the variable available to the environment
  • echo outputs a string to the console
  • printenv print the values of the enviroment variables

1

# File Operations:

  • touch: Create an empty file. Creates a new empty file named "new_file.txt."
touch new_file.txt
1
  • cp:Copy files or directories. Copies "file.txt" to a directory. The -R flag causes cp to copy the folder and its contents. Note that the folder name does not end with a slash, which would change how cp copies the folder.
cp -R file.txt ~/Documents/code
1
  • mv: Move a file or folder locally

For example, to move a file from your Downloads folder to a Work folder in your Documents folder:

mv ~/Downloads/MyFile.txt ~/Documents/Work/MyFile.txt
1

You can also change the name of the file as it’s moved:

mv ~/Downloads/MyFile.txt ~/Documents/Work/NewFileName.txt
1

# Text Manipulation:

  • cat: Display the content of a file. The cat command outputs the contents of "file.txt" to the terminal.
cat file.txt
1
  • grep: Search for Patterns in Files. This command searches for the specified "pattern" in the content of "file.txt."
grep "pattern" file.txt
1
  • nano or vim: Text Editors. Open a text editor for creating or editing files.
nano new_file.txt
1

# Permissions:

  • chmod: Change File Permissions. This command grants execute (+x) permission to the script "script.sh."
chmod +x script.sh
1
  • chown: Change File Owner. The chown command changes the owner to "user" and the group to "group" for the file "file.txt."
chown user:group file.txt
1

# Alias

Depending on what you're using the terminal for, you'll almost certainly find yourself writing the same commands over and over again. And over and over again.

This isn't good for your productivity. Or your fingers. It would be wise to create a shorter and easier-to-type alias for the commands you write the most.

For example, if you want to go to your Documents/code/MAD9112 folder without typing out the whole command on your terminal, you can create an alias mad9112 for that folder.

# Edit ~/.zshrc
